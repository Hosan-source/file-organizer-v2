## 3차 검토 수정 (타이밍 버그 1건)

| 문제 | 심각도 | 상세 |
|------|--------|------|
| 다이얼로그 표시 중 다른 작업 실행 가능 | **치명** | `_do_dup`/`_do_rename_preview` 함수가 끝나면 `_wrap`의 `finally`에서 `_is_running=False`로 풀림. 다이얼로그가 떠있는 동안 사용자가 다른 버튼 클릭 가능 → 동시 작업 발생 |

### 수정: `_keep_running` 플래그 도입

```
_do_dup 시작
  → _is_running = True (버튼 비활성)
  → 검색 수행
  → _keep_running = True  ← 다이얼로그로 넘기기 전에 설정
  → self.after(show_dialog)
  → _wrap finally 도달
  → _keep_running == True → 잠금 해제 안함 ★

다이얼로그 X 클릭 → _finish_running()
  → _is_running = False, _keep_running = False
  → 버튼 복원

다이얼로그에서 삭제 실행 → _run_thread(skip_init=True)
  → _keep_running = False (리셋)
  → 새 스레드에서 삭제 수행
  → _wrap finally → _keep_running == False → 정상 해제
```

이 구조로 **다이얼로그가 떠있는 동안 버튼이 잠긴 상태를 완벽하게 유지**합니다.
